诸神愚戏馆主后台二期与运营升级计划书

更新时间：2026-07-19

一、当前状态

馆主后台一期已完成：玩家查询、称号/诅咒列表与回收、天赋仓库查看、天赋异常扫描及可安全自动修复。

但当前后台仍有两个管理缺口：

1. 管理动作没有统一审计记录，出现误操作时难以确认是谁、何时、对谁做了什么。
2. 已回收的称号或诅咒不能在后台恢复。

二、本期目标

本期先完成“馆主后台二期：操作日志与恢复能力”。

范围：

1. 新增 admin_operation_logs 管理操作日志表。
2. Edge Function 统一写入馆主后台操作日志。
3. 称号授予、回收、恢复写入日志。
4. 诅咒下放、回收、恢复写入日志。
5. 天赋异常扫描、修复写入日志。
6. 结算撤销写入日志。
7. 馆主后台显示全站最近 50 条操作，以及当前玩家最近操作。
8. 馆主后台支持恢复指定已回收称号和诅咒。

不在本期范围：

1. 玩家公开成长时间线。
2. 赛季、报名、副本推荐与作者数据面板。
3. 自动回滚分数结算以外的任意历史状态。
4. 删除审计日志。

四、数据安全规则

1. 日志只记录邀请码 hash 于数据库内部，任何前端响应均不返回 hash 或邀请码明文。
2. 操作日志只能由 Edge Function 使用 service_role 写入，普通前端不能直写。
3. 恢复只允许馆主执行；神明仍保留现有称号/诅咒权限，但不能进入完整馆主后台。
4. 日志表只追加，不提供前端删除能力。
5. SQL 未执行时，旧功能继续可用；后台会提示日志尚未启用。

五、验收标准

1. admin 能看到全站最近 50 条后台操作。
2. 查询玩家后能看到该玩家的相关操作记录。
3. 回收和恢复指定称号/诅咒后，状态立即刷新且产生一条日志。
4. 天赋扫描、天赋修复和结算撤销会产生对应日志。
5. 非 admin 直接调用恢复或查询日志动作会被 Edge Function 拒绝。
6. 前端和后端均不向页面返回邀请码明文或 hash。

六、后续排期

二期完成后，下一优先级为“玩家成长时间线”：后台整合通关、结算、称号、诅咒和天赋事件；公开档案只展示适合公开的履历。之后再做作者数据面板、赛季与网站端报名。
